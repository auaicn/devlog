# Basic Linux Commands regarding directories

---

### pwd

```shell
$ pwd
```

print working directory 의 약자로, working directory 는 **터미널에서의 현재 위치**를 의미한다.

터미널의 입력부분에서도 현재 위치를 알 수 있지만, 이는 상대경로로 표기되는 경우가 많다.

pwd 는 현재 위치의 절대 경로를 알려준다.

### ls

```shell
$ ls
```

list 의 약자로, 현재 위치, 즉 working directory 에 있는 파일 혹은 디렉터리를 보여준다.

### cd

```shell
cd [상대경로 or 절대경로]
```

change directory 의 약자로, 입력되는 argument 에 따라서, 해당 경로로 working directory 를 변경한다.

아무런 argument 를 주지 않는경우, 현재 로그인된 유저의 home directory 로 이동할 수 있다.

# Linux Directories

---

Linux 계열의 운영체제에서는, 리눅스 재단에서 유지하고 있는 [FHS 표준](https://en.wikipedia.org/wiki/Filesystem_Hierarchy_Standard) 를 따른다고 한다.

### `/`

루트, 최상위 디렉터리로 앞으로 나오는 모든 디렉터리들을 포함하는 디렉터리이다.

### `/lib`

libraries 의 약자로, 아래 설명하게될 `/bin` 이나 `/sbin` 등의 바이너리에 공통으로 사용되는 코드들이 위치하게 된다.

### `/bin`

bin 은 binary 의 약자로, 실행 가능한 파일들을 모아두는 곳이다.

앞서 언급한 `pwd` `ls` `cd` 등의 명령어는 사실 실행가능한 파일인데, 해당 파일은 실제로 이 디렉터리에 위치하게 된다. 터미널은 명령어를 받아 해당 파일을 실행시켜주는 역할을 할 뿐이다.

헷갈릴 수 있는 점으로, `/bin` 디렉터리는 여러 군데서 발견할 수 있다.

1. `/` 루트 디렉터리 하위 

특히 OS 가 필요로 하는 바이너리들이 위치한다.

2. `/usr` 디렉터리 하위

OS 가 필요로 하기보다, 사용자들이 필요로 하는 바이너리들이 위치한다.

3. `/usr/local` 디렉터리 하위

사용자가 직접 컴파일한 바이너리들이 위치한다.

바이너리의 위치에 상관없이, 지금까지 터미널에서 커맨드를 입력할 수 있었던 것은, 터미널이 커맨드를 받아, 실제 실행가능한 파일을 위 세가지 위치에서 확인해왔기 때문이다. 위 세가지 위치는 환경변수 `$PATH` 에 등록되어있으며, 터미널은 이를 참고하여 검색을 실시한다.

```shell
$ echo $PATH
```

명령어를 터미널에 입력하면, 위 세가지 경로가 등록되어있는 것을 확인할 수 있다.

그렇다면 여러가지 PATH 중에, 바이너리가 위치한 곳을 알 수는 없을까?

아래 명령어로, PATH 중 특히 어떤 폴더에 위치하고 있는지 확인할 수 있다.

```shell
which ls
```

### `/sbin`

<p>
<U>s</U>ystem <U>bin</U>ary 의 약자로, root (super) user 에 의해 사용되는 실행가능한 파일을 모아두는 곳이다.
</p>

### `/etc` 

텍스트 기반의 설정파일들을 모아두는 곳이다.

### `/home`

Linux 계열 운영체제는 여러명의 유저등록을 지원하는데, **각 사용자를 위한** 서브 디렉터리가 위치하는 곳이다.

운영체제 로그인시, 각 사용자에따라서 환경변수 `home` 이 세팅된다.

```shell
cd ~
```

터미널 명령어 `cd` 는 ~ 를 argument 로 받게 되면, 환경변수 `home` 을 확인하고 그 위치로 `pwd` 를 이동시킨다. 간편하게 현재 로그인 된 유저의 홈 디렉터리로 이동할 수 있는 원리이다.

### `/boot`

linux kernel 이 부팅 과정에서 사용하는 파일들이 위치한다.

### `/dev`

dev 는 device 의 약자로, 하드웨어나, 드라이버와의 통신간 사용되는 파일이 위치되는 곳이다.

### `/opt`

opt 는 optional 의 약자로 말 그대로 옵션으로서 add-on 소프트웨어 등이 위치하는 곳이라고 한다.

실제 사용사례는 잘 알지 못하겠다.

### `/var`

var 는 variable 의 약자로, OS 구동간의 <U>log, cache 들이 저장</U>되는 위치이다.

Docker 사용간에는 /var/run/docker.sock 에 접근하면서 본 적 있고,

Jenkins 사용간에 /var/lib/jenkins 에 접근하면서 사용되던 것을 본적 있는 것 같다.

### `/tmp`

tmp 는 temporary 의 약자로, <U>재부팅간에도 유지되는</U> 디렉터리라고 한다.

### `/proc`

proc 은 process 의 약자로, <U>실제 디스크에는 존재하지 않는</U> 디렉터리라는 점이 특별하다. 현재 돌아가고 있는 프로세스들을 추적하기 위해 메모리에만 유지된다고 한다.
